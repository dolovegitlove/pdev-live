PDev Pipeline Documentation

Complete reference for the PDev document generation pipeline

Quick Start

1

Start a Project

/pdev start my-project

Creates a new project and opens the guided wizard

2

Run Pipeline Commands

/idea my-project

Generate documents step by step through the pipeline

3

View in PDev Live

Pipeline Commands

The PDev pipeline consists of 9 sequential phases. Each command generates a structured markdown document.

/idea ‚Üí /bmrk ‚Üí /gapa ‚Üí /innov ‚Üí /caps ‚Üí /spec ‚Üí /bmrk design ‚Üí /sop ‚Üí /pv
1 /idea

Ideation

Define your project vision, target audience, core features, and success metrics.

Output: IDEATION.md
2 /bmrk

Benchmark

Competitive analysis of similar products, their strengths, weaknesses, and market positioning.

Output: BENCHMARK.md
3 /gapa

Gap Analysis

Identify gaps in the market and opportunities your product can address.

Output: GAP_ANALYSIS.md
4 /innov

Innovation

First-principles analysis of how to build something 10x better than competitors.

Output: INNOVATION.md
5 /caps

Capabilities

Technical feasibility assessment - what can be built with available resources and technology.

Output: CAPABILITIES.md
6 /spec

Product Spec

Detailed technical specification including architecture, APIs, data models, and integrations.

Output: PRODUCT_SPEC.md
7 /bmrk design

Design System

Visual and UX benchmark - colors, typography, component patterns, and interaction design.

Output: DESIGN_SYSTEM.md
8 /sop

Development SOP

Step-by-step implementation plan with phases, tasks, and deliverables.

Output: DEVELOPMENT_SOP.md
9 /pv

Pipeline Validation

Validate all pipeline documents exist and are consistent with each other.

Output: Validation Report

Alternative Entry Points

/eval

Evaluate an existing product or business instead of starting from scratch. Generates EVALUATION.md with strengths, weaknesses, and improvement opportunities.

/pdev start [project]

Interactive wizard that guides you through project setup and runs the appropriate first command.

/pdev next [project]

Automatically runs the next command in the pipeline sequence based on which documents already exist.

/pdev status [project]

Shows pipeline progress - which documents exist, which are missing, and health scores.

Scope Levels

PDev commands can operate at different scope levels depending on your needs.

Scope Command Example Description
Industry /pdev industry "wellness tech" Macro market analysis across an entire industry
Product /pdev product myapp Full pipeline for a complete product
Feature /pdev feature "booking" Pipeline for a single feature
Component /pdev component "calendar" UI component or module level
Sprint /pdev sprint myapp phase-2 Time-boxed set of tasks
Task /pdev task 3.2 Specific task from the SOP

Frequently Asked Questions

Do I have to run commands in order?

Yes, generally. Each phase builds on the previous one. However, you can skip phases if you already have that information from another source. Use /pdev status to see which documents exist.

Can I regenerate a document?

Yes. Running the same command again will create a new version. PDev Live tracks versions so you can see the history of changes.

Where are the documents stored?

Documents are stored in your project's dev/ directory (e.g., myproject/dev/IDEATION.md). They're also synced to PDev Live for viewing and sharing.

Can I share documents with clients?

Yes. From any project page, click "Share with Client" to generate a time-limited guest link. Guests can view documents but cannot modify or create new ones.

What if I'm evaluating an existing product?

Use /eval instead of /idea as your first command. This analyzes what already exists rather than starting from a blank slate.

How do I enable Git auto-commit?

Go to Settings ‚Üí Git Sync and enable automatic commits. Add your repository paths to the whitelist. PDev documents will be committed and pushed automatically.

What's the difference between /bmrk and /bmrk design?

/bmrk (phase 2) analyzes competitor products and market positioning. /bmrk design (phase 7) specifically analyzes visual design, UX patterns, and creates a design system.

Can I run PDev on a remote server?

Yes. PDev Live supports multiple server origins. Configure servers in Settings ‚Üí Servers. Documents from any connected server will appear in your project library.
